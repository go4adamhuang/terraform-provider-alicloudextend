## Project Overview

Extended Terraform provider for Alibaba Cloud — covering resources not provided by, or rewritten from, the official `aliyun/alicloud` provider. Built with [Terraform Plugin Framework](https://github.com/hashicorp/terraform-plugin-framework).

Module: `github.com/go4adamhuang/terraform-provider-alicloudextend`

## Commands

```bash
make build      # compile provider binary
make install    # build + install into ~/.terraform.d/plugins/ for local testing
make test       # unit tests
make testacc    # acceptance tests (requires real AliCloud credentials, sets TF_ACC=1)
make testacc-one TEST=TestAccFoo  # run a single acceptance test
make tidy       # go mod tidy
make fmt        # go fmt + goimports
make lint       # golangci-lint
make docs       # generate docs/ via tfplugindocs
make clean      # remove binary
```

## Architecture

```
main.go                          # entry point, registers provider address
internal/provider/
  provider.go                    # AliCloudProvider: Schema, Configure, Resources(), DataSources()
  helpers.go                     # strPtr / int64Ptr / uint64Ptr
  resource_<name>.go             # one file per resource
examples/
  provider/main.tf               # example provider block
  resources/<resource>/          # example .tf per resource
docs/                            # generated by tfplugindocs (do not hand-edit)
```

## Provider Credentials

Priority order (lowest → highest): env vars → CLI profile → explicit provider block.

| Field | Env var |
|---|---|
| `access_key_id` | `ALICLOUD_ACCESS_KEY_ID` |
| `access_key_secret` | `ALICLOUD_ACCESS_KEY_SECRET` |
| `region` | `ALICLOUD_REGION_ID` |
| `profile` | `ALICLOUD_PROFILE` |

Profile loads from `~/.aliyun/config.json` (standard Alibaba Cloud CLI format).

## Adding a Resource

1. Create `internal/provider/resource_<name>.go` implementing `resource.Resource`.
2. Register the constructor in `provider.go` → `Resources()`.
3. Add `examples/resources/alicloudextend_<name>/resource.tf`.
4. Add the AliCloud SDK package to `go.mod` via `go get` + `make tidy`.

### Resource file skeleton

```go
package provider

import (
    "context"
    "github.com/hashicorp/terraform-plugin-framework/resource"
    "github.com/hashicorp/terraform-plugin-framework/resource/schema"
)

type FooResource struct{ client *ClientConfig }

func NewFooResource() resource.Resource { return &FooResource{} }

func (r *FooResource) Metadata(_ context.Context, req resource.MetadataRequest, resp *resource.MetadataResponse) {
    resp.TypeName = req.ProviderTypeName + "_foo"
}

func (r *FooResource) Schema(_ context.Context, _ resource.SchemaRequest, resp *resource.SchemaResponse) {
    resp.Schema = schema.Schema{Attributes: map[string]schema.Attribute{}}
}

func (r *FooResource) Configure(_ context.Context, req resource.ConfigureRequest, resp *resource.ConfigureResponse) {
    if req.ProviderData == nil { return }
    r.client = req.ProviderData.(*ClientConfig)
}

func (r *FooResource) Create(ctx context.Context, req resource.CreateRequest, resp *resource.CreateResponse) {}
func (r *FooResource) Read(ctx context.Context, req resource.ReadRequest, resp *resource.ReadResponse) {}
func (r *FooResource) Update(ctx context.Context, req resource.UpdateRequest, resp *resource.UpdateResponse) {}
func (r *FooResource) Delete(ctx context.Context, req resource.DeleteRequest, resp *resource.DeleteResponse) {}
```

## AliCloud SDK

Add per-service packages as needed, e.g.:

```bash
go get github.com/alibabacloud-go/<service>-<version>/<version>
```

Use `ClientConfig.AccessKeyID` / `AccessKeySecret` / `Region` when initialising SDK clients inside `Configure()`.
