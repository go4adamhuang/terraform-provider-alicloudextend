# alicloudextend_esa_rate_plan_instances

Lists ESA rate plan instances. Returns the first matching instance_id for easy use in other resources.

## Example Usage

```terraform
# Find the first online Premium (vipplan_intl / high) plan instance
# that still has remaining site quota.
data "alicloudextend_esa_rate_plan_instances" "premium" {
  # Defaults below are pre-set; explicit here for clarity.
  plan_name_en              = "vipplan_intl"
  plan_type                 = "high"
  status                    = "online"
  check_remaining_site_quota = true
}

output "premium_instance_id" {
  value = data.alicloudextend_esa_rate_plan_instances.premium.instance_id
}

output "all_premium_instances" {
  value = data.alicloudextend_esa_rate_plan_instances.premium.instances
}
```

## Schema

### Optional

- `check_remaining_site_quota` (Boolean) Only return instances with remaining site quota. Defaults to true.
- `plan_name_en` (String) Filter by plan name (e.g. vipplan_intl). Defaults to "vipplan_intl".
- `plan_type` (String) Filter by plan type. Valid values: normal, enterprise. Leave unset to return all types.
- `status` (String) Filter by plan status: online, offline, disable, overdue. Defaults to "online".

### Read-Only

- `instance_id` (String) The instance_id of the first matching plan instance.
- `instances` (Attributes List) All matching plan instances. (see [below for nested schema](#nestedatt--instances))
